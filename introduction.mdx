## What is Ghoten?

Ghoten is an [OpenTofu](https://opentofu.org/) fork that adds one opinionated feature: a native `oras` backend for storing state in OCI registries (like GitHub Container Registry), without extra services.

We built it for teams that already trust container registries and want fewer moving parts in Terraform/OpenTofu state management. Instead of running a custom HTTP backend, you can reuse registry auth, permissions, and auditing you already have.

<Note>
**Name origin:** *Ghoten* blends **G**it**H**ub and Open**T**o**f**u, with a nod to [Goten](https://dragonball.fandom.com/wiki/Goten) from *Dragon Ball Z*.
</Note>

## Why This Approach?

For teams already using OCI registries:

- **Unified infrastructure**: Reuse the same registry, auth, and permissions you use for containers
- **Battle-tested storage**: OCI registries are designed for versioned, immutable artifacts
- **Simpler operations**: Fewer services to secure, monitor, and maintain
- **Native tooling**: Works with standard registry tools and policies

## Quick Example

Store your OpenTofu state in GitHub Container Registry:

```hcl
terraform {
  backend "oras" {
    repository = "ghcr.io/acme/infra-state"
  }
}
```

Authenticate and initialize:

```bash
echo "$GITHUB_TOKEN" | docker login ghcr.io -u YOUR_GITHUB_USER --password-stdin
./ghoten init
./ghoten plan
```

## Limitations

Before you start:

- Most production validation is on GitHub Container Registry (GHCR); other OCI registries are expected to work but are less battle-tested
- If you need advanced backend workflows (multi-region replication policies, custom APIs), dedicated backend platforms may be a better fit
- This is an upstream-tracking fork of OpenTofu focused solely on the ORAS backend
